Price filter V1 — Contextual (Buy/Rent) · เชียงใหม่
Heuristics applied in this design TH-09 Mobile-first 390px frame; filter placed at thumb-reach top zone (not buried in a modal bottom-sheet requiring a second tap) — one primary interaction surface, immediately visible.

TH-11 Filter sheet avoids over-density on the browse/landing level: one segmented toggle + one slider + presets — not parallel buy/rent price rows. Complexity reserved for search-result/decision screens.

TH-12 Trust-blue (#1f5fad) token drives the mode toggle active state, the slider fill, and the apply CTA — consistent brand color register (blue = safe/credible), not a decorative hue.

CONV-08 List-first with prominent filter as the primary search surface; the price chip in the filter row shows the active range summary at a glance so users can confirm their constraint without reopening the sheet.

CONV-09 Price always carries the "ราคาเสนอขาย" (asking) label — never presented as transaction value. The rent mode relabels to "ค่าเช่า/เดือน" so the framing is always honest.

B2-F11 / A2-F11 Thai number formatting in presets uses ล้าน (millions) for Buy and พัน/หมื่น (thousands) for Rent — matching the real Chiang Mai price distribution: houses ฿1–9M (A2-F6), long-term rents ฿7k–45k/mo (A2-F11). Choice overload is reduced by offering only 4–5 presets calibrated to this market's real range.

B1-F11 Segmented ซื้อ/เช่า control doubles as a mode-aware deal-type filter (not a redundant second chip) — the SAME toggle in the filter row and the sheet stays in sync, preventing the "two controls that can disagree" anti-pattern that causes confusion on Thai property portals (DDProperty/LivingInsider baseline, B2-F16).
